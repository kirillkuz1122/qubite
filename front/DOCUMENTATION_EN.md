## üöÄ Core Concepts

### 1. View Management

The application operates as a Single Page Application (SPA). Key containers are:

- `#landing-view`: The public home page.
- `#workspace-view`: The private user dashboard (hidden by default).

**Key Functions in `app.js`:**

- `switchToWorkspace()`: Hides the landing page, shows the Workspace, and opens the dashboard view.
- `switchToLanding()`: Returns the user to the landing page.
- `ViewManager.open(viewName)`: Dynamically swaps content within the Workspace (Dashboard, Tournaments, etc.).

### 2. Dynamic Rendering

Content inside `#workspace-content` is generated by renderer functions:

- `renderDashboard()`: Generates HTML for the dashboard home.
- `renderTournaments()`: Generates HTML for the tournament browser.
- `renderTournamentList(data)`: Helper function to render tournament cards based on a data array.

### 3. Theme System (Dark/Light Mode)

Uses CSS variables defined in `:root` and `.light-theme`.

- `toggleTheme()`: Toggles the theme and persists the choice in `localStorage`.

### 4. Global Utilities

Defined at the top of `app.js` for convenience:

- `Loader`: Show/hide the global loader (`Loader.show()`, `Loader.hide()`).
- `Toast`: Popup notifications (`Toast.show(title, desc, type)`).
- `Haptic`: Mobile vibration feedback.
- `Sound`: UI sound effects playback.
- `MegaSearch`: Smart search utility with layout correction and fuzzy matching.

## üõ† Extension & Debugging Guide

### How to add a new section to the Workspace?

1. In `index.html`, find `<nav class="sidebar__nav">` and add a new link with `data-view="my-section"`.
2. In `app.js`, inside `ViewManager.open(viewName)`, add a conditional branch:
    ```javascript
    else if (viewName === "my-section") {
        this.content.innerHTML = renderMySection();
    }
    ```
3. Implement the `renderMySection()` function that returns an HTML string.

### Troubleshooting

- **Layout/Styling issues**: Check `css/styles.css`. Look for color variables at the top (`--bg`, `--fg`, `--accent`).
- **Navigation logic**: Check the `ViewManager` object in `app.js`.
- **Modals & Auth**: Modal templates are stored in the `DYNAMIC_MODALS_HTML` constant in `app.js`. Launch logic is in `openModal(id)`.

## üìå Data

Tournament data is currently hardcoded in the `TOURNAMENTS_DATA` array within `app.js`. This is the primary location for implementing Backend API fetches in the future.
