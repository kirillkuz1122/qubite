# Инструкция по созданию презентации в NotebookLM

## Что загрузить в NotebookLM как источники

### Обязательные источники (загрузить все):

1. **Пояснительная записка (PDF)** — файл `docs/explanatory-note-typst/output/individual-project-note.pdf`
   - Это главный документ. Из него берутся тема, цель, гипотеза, задачи, содержание глав.

2. **Текст защиты** — файл `docs/explanatory-note-typst/defense-speech.md`
   - Готовая речь по слайдам. NotebookLM будет использовать его как основу для структуры презентации.

3. **Документация по архитектуре** — файл `docs/architecture.md`
   - Техническая архитектура: схема, модули, потоки данных. Для слайда про архитектуру.

4. **Документация по ролям и продукту** — файл `docs/product-and-roles.md`
   - Роли, сценарии, функции платформы. Для слайдов про функционал и роли.

### Опциональные (если хочешь более насыщенную презу):

5. **README.md** (корень проекта) — общее описание проекта.

6. **TODO.md** — roadmap и границы MVP, для слайда про развитие.

7. **Скриншоты** (4 файла из `docs/explanatory-note-typst/assets/screenshots/`):
   - `home.png` — главная страница
   - `tournaments.png` — раздел турниров
   - `profile-analytics.png` — профиль и аналитика
   - `team-analytics.png` — командная аналитика
   - Эти скриншоты нужны для слайда с демонстрацией. Загружай как изображения.

## Промпт для NotebookLM

Скопируй этот промпт и вставь в NotebookLM после загрузки источников:

---

```
Создай презентацию для защиты школьного индивидуального проекта на основе загруженных источников.

ВАЖНО: все формулировки темы, цели, гипотезы, задач, объекта и предмета исследования бери СТРОГО из пояснительной записки. Не перефразируй и не придумывай свои — иначе снимут баллы за несоответствие.

Данные из пояснительной записки (для точности):
- Тема: «Разработка веб-платформы для автоматизации проверки знаний: от школьных заданий до олимпиад»
- Автор: Кузмичев Кирилл Павлович, 8 «Б» класс
- Руководитель: Арабаджи Андрей Александрович
- Учебное заведение: ГБОУ «Челябинский областной многопрофильный лицей-интернат для одарённых детей»
- Город: Челябинск, 2026
- Продукт: MVP-прототип веб-платформы Qubite

Структура презентации (15 слайдов):

Слайд 1 — Титульный слайд
- Полное название учебного заведения
- «Проектная работа по информатике»
- Тема работы (ТОЧНО как в записке)
- Ученик: Кузмичев Кирилл Павлович, 8 «Б» класс
- Руководитель: Арабаджи Андрей Александрович
- Челябинск, 2026

Слайд 2 — Актуальность и проблема
- Проблема ручной проверки знаний в школе
- Затраты времени учителя на техническую организацию
- Для учеников — результаты не видны сразу, олимпиадный формат сложен
- Актуальность единой цифровой платформы

Слайд 3 — Идея проекта, объект и предмет
- Идея: веб-платформа с турниром как главным объектом
- Объект исследования — цифровые средства организации и проверки учебных заданий
- Предмет исследования — архитектура, функциональные возможности и способы практической реализации веб-платформы

Слайд 4 — Гипотеза, цель, задачи
- Гипотеза (ТОЧНО из записки)
- Цель (ТОЧНО из записки)
- 5 задач (ТОЧНО из записки)
- Можно добавить таблицу SMART из записки

Слайд 5 — Методы работы и тип проекта
- 5 методов из записки
- Тип: прикладной, практико-ориентированный с элементами конструкторской разработки
- Продукт: MVP-прототип веб-платформы Qubite

Слайд 6 — Анализ аналогов
- Таблица: онлайн-формы, сервисы викторин, электронные таблицы, олимпиадные системы
- Преимущества и ограничения каждого
- Вывод: выбран гибридный подход

Слайд 7 — Архитектура решения
- Схема: Браузер → Клиентский JS → Express-сервер → SQLite → Инфраструктура
- Клиент-серверная архитектура
- Монолитное web-приложение

Слайд 8 — Технологии
- HTML5, CSS3, JavaScript (клиент без фреймворков)
- Node.js, Express (сервер и REST API)
- SQLite (БД)
- Cloudflare Turnstile, OAuth, Telegram Bot API
- Nginx, PM2, Cloudflare (production)

Слайд 9 — Реализованный функционал
- Список ключевых функций из записки
- Регистрация, OAuth, турниры, задачи, рейтинг, импорт, Telegram-бот
- Ежедневные задания

Слайд 10 — Роли пользователей
- Таблица с 5 ролями и их возможностями
- user, organizer, moderator, admin, owner

Слайд 11 — Безопасность
- scrypt, сессии, email-подтверждение, CAPTCHA, rate limiting, аудит
- Защита owner-роли, режим обслуживания
- Конфигурации для reverse proxy и firewall

Слайд 12 — Демонстрация платформы
- 4 скриншота: главная, турниры, профиль+аналитика, командная аналитика
- Если можешь вставить загруженные скриншоты — вставь

Слайд 13 — Границы MVP и развитие
- Что реализовано vs что в планах
- Code runner, AI-проверка, апелляции, визуальные задачи — в roadmap
- Улучшение интерфейсов организатора и модератора

Слайд 14 — Заключение
- Цель достигнута
- Гипотеза подтверждается
- Все 5 задач решены (перечислить кратко)
- Практическая ценность

Слайд 15 — Спасибо за внимание
- «Спасибо за внимание»
- «Готов ответить на ваши вопросы»

Стиль:
- Строгий, академический. Без мемов, анимаций и картинок из интернета.
- Шрифт без засечек (Arial, Calibri или аналог).
- Минимум текста на слайдах — ключевые тезисы, таблицы и схемы.
- Цветовая схема: нейтральная (белый фон, тёмный текст, один акцентный цвет).
- На каждом слайде должен быть заголовок.
```

### Порядок действий:
1. Открой [NotebookLM](https://notebooklm.google.com/)
2. Создай новый ноутбук
3. Загрузи источники из таблицы выше (как минимум первые 4)
4. Скопируй промпт из раздела выше
5. Вставь промпт и отправь
6. Проверь, что на слайдах тема, цель и гипотеза совпадают с пояснительной запиской ДОСЛОВНО
